List RegUser 報名人數管理

Description

報名者管理, 列出所有活動的參加者, 並加以編輯、修改參加者的狀態, 使用select選單選取活動.



Navigation

使用此功能: Components -> EventList -> 報名人數管理

User Access Level

Super Administrator Administrator Manager

Toolbar Icons

使用CheckBox選擇想要編輯的使用者

統計圖:當活動開放問卷時, 才會顯示此問卷, 以圓餅圖顯示問卷調查結果.

新增使用者: 手動加入使用者

TXT: 將訂閱news letter的名單, 列成TXT文件

CSV: 將參加者名單列成CSV文件

HTML: 將參加者名單以開啟新視窗方式輸出成HTML文件

加入黑名單: 將使用者列為黑名單, 往後使用者在參加活動時會發出警告訊息, 並說明哪個活動未參加.

取消報名: 將使用者取消報名.

FILTER MAIL: 寄信給相關活動的參加者.

MAIL TO ME: 寄出活動信件給自己

SENDMAIL: 寄出信件給被勾選的參加者

Help: 連結說明頁面.

表單:

序號 使用者報名序號

姓名: 使用者姓名

電子信箱: 使用者信箱.

ALL-FUTRUE:(參加過得活動/尚未參加的活動)

公司名稱: 使用者公司名稱(前台選填).

部門/職稱: 使用者部門與職稱(前台選填).

社群: 使用者參與的社群(選填).

報名日期: 使用者的報名日期.

USE VIPCODE: 報名時所使用的VIP CODE.

JOIN STATE: 使用者目前狀態(參加,取消,由管理者加入,由管理者取消)

備註: 管理者對於該使用者的備註, 可在編輯使用者中編輯此欄位.

HTML

欲列印檔案時, 將紙張大小設定為A4, 否則會有表單破圖的情況發生.

若想增加輸出的參加者, 可使用新增使用者來增加避免檔案輸出時有表單異常的情況, 使用者資料可在新增後進行編輯.

SEND MAIL

sendmail有兩種寄件方式

A:由使用活動列表勾選使用者:寄件者為活動聯絡人

B:使用區域或分類列表勾選使用者:寄件者為登入者

信件寄出時會提示寄件人姓名

新增使用者

使用者姓名與電子信箱:需要填入使用者的email以及姓名, 至少填入email

寄信給使用者:若勾選會將報名成功信件寄給使用者,反之則不會

姓名: 被新增的使用者姓名

信箱: 被新增的使用者信箱

發送報名信件:決定新增後是否將報名信件寄出給該使用者

select選單

選擇活動:選擇後顯示該活動的所有使用者

選擇分類:選擇後顯示該分類的所有使用者 若有選擇活動必須先點重置, 將select選單重置

選擇區域:選擇後顯示該區域的所有使用者 若有選擇活動必須先點重置, 將select選單重置

選擇信件:使用mail to me 或 sendmail時所使用

選擇使用者狀態:僅在選擇活動後出現, 可針對使用者狀態在做一次篩選

使用者狀態

位於使用者姓名後面會有四種狀態圖示, 若沒有任何圖示則是無特殊情況的使用者
初次參加活動的使用者
取消參加活動的使用者
在本次活動被列為黑名單的使用者
訂閱電子報的使用者

FILTER MAIL

在選取活動之後, 系統會自動帶入該活動所設定的活動分類以及活動區域

選擇信件後, 點選FILTER MAIL 系統會寄信給 參與過相關活動的使用者, 並且排除掉已參加該活動的使用者
